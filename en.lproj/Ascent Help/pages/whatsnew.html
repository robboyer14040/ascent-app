New Features in Ascent

   

Version 1.7

  • Added TCX export
  • Added support for direct syncing of Garmin Vista 60CSx
  • You can now edit the Activity, Equipment, Disposition, Weather, Effort, and Training lists in the "Info" pane of the main browser (there is now an "Edit..." menu item at the end of each of the popups)
  • Keywords are now presented as a popup list (which you can also edit, of course!)
  • Date format now has several options settable in "General" preferences
   

Version 1.6

  • Added speed, pace, gradient, and cadence zone support; ranges are settable in a new preference pane
  • Paths in the Detailed Map View can now be color-coded by zone
  • Laps can be highlighted in the Detailed Map View
  • New zone types can be displayed in the Activity View
  • Zone type can be selected in the Activity View Statistics HUD
   

Version 1.5

  • Extended heart rate zone system to include 5 zones, as well as different methods of calculating zone ranges (Kervonen, Zoladz, Classic, and Custom)
  • Activity data can now be exported to comma or tab-separated text files
  • Cut/Copy/Paste can now be used to paste tab-delimited values into an external application such as Excel
  • Now simultaneously display 'speed' and 'pace' in Data HUD in the Activity View
  • Arrow keys will can now be used to fine-tune end of the selection in the Activity View (for the Statistics HUD), holding the OPTION key while pressing the arrow keys allows fine-tuning of the start of the selection.
  • Replaced Activity View "Data HUD" with new, easier-to read version that has the same style as the statistics HUD
   

Version 1.4

  • Laps can now be displayed (by themselves) in the detailed activity view
  • "Activity Statistics Heads-Up Display (HUD)" now available in the detailed activity view. Shows data for track, lap, or selected region. Option-drag to select a region, and the HUD will dynamically update.
  • Markers now display in "active time" view of the detailed activity view.
   

Version 1.3

  • Major re-write of the moving-time calculation algorithm - the advanced preference "Minimum speed for moving time calculations" now functions correctly. A value between 1.0mph and 1.3mph (1.6km and 2.0km) seems to work best.
  • Added mandatory speed filtering to remove spikes
  • Speed values in the Activity Data view can now be edited
  • "Stats" pane in browser now displays lap statistics when a lap is selected
  • In the activity view, x and y axis values now show more or less values depending on size of the window -- in other words, the number of tick marks changes as you resize the window
  • Activity view graphs now stretched to fit in available space with no empty area at the end
  • Selecting a lap in the browser now highlights the lap's section in the mini-profile view (of the browser)
  • XS"Save selected...(activity)" (in file menu)
  • X"Email selected...(activity)" (in file menu, requires OSX Mail.app to be configured)
  • Contextual menus in main browser, activity detail, and map detail views (invoked by right mouse button on multi-button mouse, control-click on 1-button mouse)
  • "center map at click point" available in contextual menu of detailed map view
  • Store most view settings persistently: browser view, activity detail view, map views
  • Improvements to the map detail view, so that zoom and other controls work like standard map applications
  • Improved "Notes" entry in the main browser, using a dedicated editing panel that supports spell checking, undo, etc.
  • Added hot keys in map detail view, such as "I" for zoom in, "O" for zoom out
  • Mouse scroll wheel in the map detail view now performs zooming
  • .tcx/.hst import now imports "Notes" into activitites
   

Version 1.2

  • Now show pace in activity view HUD
  • Added preference for "enable auto-split" and "auto split threshold". This feature allows tracks with every long in-active periods to be split into separate tracks
  • Added general preference for universal (24-hour) time display, instead of am/pm
  • Added appearance preferences to set colors for different text items in the browser
  • Added speed/pace selector to activity view
  • Modified activity view to display pace graph, peaks, etc (still need to do heads-up display)
  • Modified detailed map view HUD to display speed + pace
  • Browser now supports "flat" displays of all activites, weeks, or months. All columns can be sorted.
  • During import, any time gap longer than 1 hour will be split into a new activity/track. This helps when you forget to reset the Garmin GPS in between workouts.
  • Laps now appear in the main browser, and the previous "Laps" tab has been removed. When a Lap row is selected in the browser, it will highlight in the map overview.
  • Much work to make sure data values align with Garmin values
  • Added "altitude threshold" preference to eliminate extraneous noise in climb/descent readings. Value differences less than the preference value will be ignored when computing climb or descent values.
  • Ascent now can display data from indoor activies, e.g. activities with no GPS and/or altitude data
  • Heart rate values of 0 are now not included in averages
  • Ignore cadence values of "255" (signal from Edge that cadence is not available)
  • Added code to fix missing GPS values
   

Version 1.1

  • Added the ability to import .tcx and .hst files (these are the files that are exported from the Garmin Training Center application).
  • Added the ability to import and export .gpx files (this is a standard GPS Exchange Format, supported by many companies).
  • Major work on the main browser:
    • you can right-click (or control-left-click) any column header and choose which columns are displayed in the browser.
    • columns can be re-ordered by dragging their headers (all except the first column, which can never be moved). -- column ordering and widths are stored in and retrieved from the document file.
  • Added "pace" display options (one of the columns that can be added to the browser). Pace displays minutes/mile or minutes/kilometer.
  • Added "moving time" display options in the browser. In these cases, data points for speed, duration, and pace are only used in calculations if the speed was above a certain threshold settable in Advanced Prefs. This eliminates stopped or nearly stopped time from these calculations.
  • Added "total time in heart rate zone" displays in the browser.
  • Cadence values of 0 are now ignored in averages.
  • Window sizes (for each window type) are now stored in the preferences file, so they are remembered across invocations of Ascent.
  • Added a "Number of points to average" control in the Detailed Activity view, which is used by the "Average" graphs.
  • Changed some menubar menu items to have more descriptive names.
